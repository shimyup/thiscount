# 타워 대신 적용할 레벨업 요소 제안 (Build 121 업데이트)

> **Build 121 결정 반영**: 주력 방안(후보 1 — 픽업 반경 링 + 헌터 아바타)
> 과 보조 방안(후보 2 — 스탬프 앨범 ▸ **아이템형 이모지 스탬프** 로 변경)
> 을 동시에 구현 완료.
>
> **Build 122 추가**: 정적 플래그 아바타 → **레벨 기반 진화 캐릭터 이모지**
> 로 한 번 더 진화. 플래그는 아바타 중앙에서 하단 pill(🇰🇷 Lv 7) 로 이동.
> "헌터" 명칭 대안도 5개 추천 (명칭 선택은 사용자 결정 대기).

현재 "타워" (`activityScore.tier` + `towerFloors` + 타워 스킨) 는 Build 101
이전 "슬로우 소셜 펜팔" 시절의 진급 메타포. Thiscount 가 **픽업 중심
보물찾기** 로 전환된 지금, 타워의 "층이 쌓인다" 은유가 더 이상 게임 루프
와 직결되지 않는다.

이 문서는 타워를 **완전 교체 / 부분 교체 / 보완** 할 수 있는 5가지 후보를
제시하고, 각각의 구현 비용·효과·리스크·기존 자산 활용도를 비교한다.

---

## 현재 타워의 역할 정리 (교체 대상 범위)

| 기능 | 현재 구현 | 교체 필요 여부 |
|------|-----------|----------------|
| 지도 위 내 위치 마커 | `_MyTowerMarker` (건물 모양 + 층 수 표시) | 교체 or 단순화 |
| 지도 위 다른 유저 마커 | `_buildMapTowerMarkers` (클러스터링) | 유지 (사회적 표식) |
| 티어 뱃지 | `activityScore.tier` (beginner/casual/regular/experienced) | 교체 or 병행 |
| 타워 높이 | `towerFloors` (활동 점수 기반 정수) | 의미 재정의 |
| Premium 스킨 | `premiumFeature4` "타워 커스텀 색상" | 다른 커스텀으로 이동 |
| 프로필 "타워 진행바" | `_buildTowerProgressCard` | 교체 or 병행 |

**핵심 결정**: 지도 "다른 유저 존재" 표시는 사회적 fabric 으로 유지하되,
**내 레벨·진급 비주얼은 픽업 메타포로 전환** 하는 것이 헌트 포지셔닝과
정합.

## 🥇 후보 1 — **픽업 반경 링 + 헌터 아바타** (주인공 승격)

### 컨셉
Build 120 에서 추가한 픽업 반경 링 자체가 **레벨의 시각적 결과물**. 링
중앙엔 작은 원형 헌터 아바타 (🗺/🎯/🧭 이모지 + 국가 플래그). 링 크기 ·
색 = 레벨 · 티어. 타워 마커 전면 교체.

### 시각
```
       ╭─────────────╮
       │             │
       │      🎯     │   ← 내 아바타 (작게)
       │   🇰🇷        │
       │             │
       ╰─────────────╯   ← 반경 링 (확대 = 레벨업)
```

### 레벨업 → 링 확장 애니메이션
레벨업 시 200–400ms ease-out 으로 링이 +10m 만큼 확대. 동시에 링 색이
한 번 깜박 (alpha 0.8 → 0.3). **직접 체감**.

### 프로필
"내 타워" 섹션 → "내 헌트 프로필". 중앙에 큰 원형 프리뷰 (사용자 플래그 +
레벨 숫자 + 현재 반경). 하단에 HuntWalletCard 의 반경 진행바 재사용.

### 구현 비용 · 기존 자산 활용
- ✅ 반경 링 이미 존재 (Build 120)
- ✅ 아바타용 국가 플래그 이미 있음
- ⏱ `_MyTowerMarker` 교체: 건물 → 원형 아바타 (~80 LOC)
- ⏱ 프로필 `_buildTowerProgressCard` → `_buildHunterProfileCard`
- ❌ 다른 유저 타워 마커는 유지 (사회적 표식 용도)
- ❌ Premium "타워 커스텀 색상" → "헌터 아바타 배경" 으로 리포지션
- **총 비용**: Medium (2–3 빌드)

### Pros
- 이미 핵심 시각(링) 이 기존에 있어 재사용 극대
- "레벨업 = 반경 확대" 일체화. 가장 강력한 게임플레이 ↔ 시각 정합
- 타워라는 정적 은유 대신 **유저 본인 중심** 의 동적 시각

### Cons
- 지도에서 타워가 사라지면 심미적 아이덴티티 일부 손실 (기존 빌드 사진
  썸네일 재촬영 필요)
- 다른 유저 타워 마커는 유지하면서 내 것만 교체 → 마커 유형 혼재

## 🥈 후보 2 — **아이템형 이모지 스탬프** ✅ Build 121 구현

### 최종 컨셉 (변경됨)
스탬프 앨범 그리드(50칸)는 "도감 채우기" 느낌이 강해 **헌트 본질(주워 쓰는
혜택)** 과 거리 있음. 대신 "주워 모은 **장비**" 인벤토리 형식으로 전환.
마일스톤 레벨 5개(Lv 2 / 5 / 10 / 25 / 50) 에 각각 **대표 아이템 이모지**
를 매핑해 헌터의 장비함처럼 노출.

### 아이템 매핑
| 레벨 | 이모지 | 의미 |
|------|--------|------|
| 2 | 🎯 | 조준 — 첫 마일스톤 |
| 5 | 🧭 | 나침반 |
| 10 | 🗺 | 보물 지도 |
| 25 | 🎒 | 여행자 배낭 |
| 50 | 👑 | 전설의 왕관 |

### 시각 (구현 완료)
- **프로필 HuntWalletCard** 반경 바 바로 아래 "헌터 아이템" 섹션 (5칸 가로
  슬롯). 획득 = 풀컬러 이모지 + 골드 테두리 + `Lv N`, 미획득 = 회색 이모지
  + 🔒 + 툴팁 "Lv N 에 해금".
- **지도 내 아바타 좌상단** 에 가장 높이 획득한 아이템 이모지 작게 (12px).
  "지금까지 주운 대표 장비" 를 한 눈에.

### 구현 위치
- `AppState`: `_hunterItemEmojis` Map + `hunterItemEmoji(level)` +
  `earnedHunterItemLevels` + `latestHunterItemEmoji` getter
- `HuntWalletCard._buildHunterItems()` — 프로필 5-슬롯 줄
- `_HunterItemSlot` (신규 위젯) — 개별 슬롯
- `_MyTowerMarker` → `milestoneItemEmoji` 파라미터로 아바타 좌상단 노출

### 왜 "아이템" 인가 (앨범 그리드 대비)
- **수량 적음** — 5개 vs 50개. 과도한 수집 압박 없음
- **의미 명확** — 각 이모지가 도구/장비로 읽혀 "줍기" 메타포 직결
- **지도와 연동** — 가장 높은 아이템을 내 아바타 뱃지로 재사용 가능
- **Premium 충돌 없음** — 이미 골드 링·커스텀 배경으로 차별화 있어
  스탬프 테두리 같은 추가 장치 불필요
- **구현 비용 ↓** — 이모지만 쓰므로 디자인 에셋 제작 불요

## 🥉 후보 3 — **헌터 백팩**

### 컨셉
지도 마커 = **배낭을 멘 작은 캐릭터**. 백팩 크기/색 = 레벨. 픽업할 때마다
백팩이 잠시 반짝. 프로필 = 백팩 인벤토리 (최근 주운 편지 썸네일).

### 시각
🎒 크기가 점점 커지는 애니메이션. 레벨업 시 백팩 폭 +5%.

### Pros
- "줍기" 메타포와 1:1 매핑 (아이템을 줍어 배낭에 넣는다)
- "내 인벤토리" 로서 Hunt Wallet 과 자연스럽게 연결
- 귀여운 시각 — 유저 친화적

### Cons
- 캐릭터 에셋 디자인 필요 (일러스트 또는 이모지 조합 제한적)
- Flutter 에서 세밀한 캐릭터 애니메이션 구현 부담

### 구현 비용
- ⏱ 캐릭터/백팩 에셋 필요 (외부 일러스트레이터 또는 Lottie)
- ⏱ 마커 애니메이션 (~100 LOC)
- **총 비용**: Medium-High (2–4 빌드 + 에셋)

## 🎮 Build 122 — 진화 캐릭터 이모지

Build 121 에서는 아바타 중앙을 **국가 플래그** 로 두었음. "내 정체성 = 국적"
이라는 의미는 있었지만 **레벨이 올라도 변하지 않음** → 성장 체감이 약했음.

Build 122 에서 교체:

### 10-티어 진화 목록 (5레벨씩)

| 레벨 | 이모지 | 의미 |
|-----|--------|------|
| 1–4 | 🧑 | 평범한 사람 (시작) |
| 5–9 | 🚶 | 걷기 시작 |
| 10–14 | 🏃 | 달리는 사람 |
| 15–19 | 🧗 | 등반가 |
| 20–24 | 🕵 | 탐정 |
| 25–29 | 🥷 | 닌자 |
| 30–34 | 🧙 | 마법사 |
| 35–39 | 🦸 | 영웅 |
| 40–44 | 🧝 | 요정·엘프 |
| 45–50 | 👑 | 전설의 왕관 |

### 선정 기준
1. 모두 **사람/인격체 이모지** — 도구는 이미 헌터 아이템 슬롯에서 표현
2. **초보 → 활동적 → 기술 → 달인 → 전설** 의 명확한 서사 아크
3. **단일 코드포인트**만 사용 — ZWJ 결합(🧑‍🚀 등) 피해서 OS/렌더러 간 호환성 확보
4. **젠더 중립** — 특정 성별 암시 없음

### 플래그는 어디로?
- 중앙에서 **하단 pill** 로 이동 → "🇰🇷 Lv 7" 결합 표시
- 국적 정보는 유지하되 **주인공은 진화 캐릭터**

### Brand 계정
- Brand 는 XP/레벨 밖이므로 항상 **👑 고정**
- 하단 pill 은 "🇰🇷 👑"

### 구현 위치
- `AppState._characterEmojisByTier` (10-tier 배열)
- `AppState.characterEmojiForLevel(level, {isBrand})` static helper
- `AppState.currentCharacterEmoji` getter (UI 편의용)
- `_MyTowerMarker.characterEmoji` 파라미터 → 중앙 Text 에 26px 렌더
- 기존 플래그는 하단 pill 로 이동 (🇰🇷 Lv 7 결합)

## 📈 Build 126 — 레벨 해금 재분배 (Lv 25–45 공백 해소)

Build 125 기준 Lv 25–45 구간이 "긴 공백" 이라 리텐션 슬럼프 우려. 세
시스템(아이템·동행·악세사리) 의 해금 레벨을 재배치해 **거의 매 레벨마다
뭔가 열리는** 조밀한 진행 곡선으로 전환.

### 변경 요약

| 시스템 | 이전 | 이후 |
|--------|------|------|
| 레터 아이템 | Lv 2·5·10·25·50 (5슬롯) | **Lv 2·5·10·20·35·50 (6슬롯, ⚡ 추가)** |
| 마일스톤 모달 | Lv {2,5,10,25,50} | **Lv {2,5,10,20,35,50}** |
| 동행 | Lv 3·8·18·28·38·48 | **Lv 3·8·15·25·35·45** |
| 악세사리 | Lv 4·12·20·30·40·50 | 변경 없음 |

### Build 126 이후 해금 스케줄

```
Lv 2  🎯 item        Lv 25 🦉 companion
Lv 3  🐕 companion   Lv 30 💎 accessory
Lv 4  🎩 accessory   Lv 35 ⚡ item + 🐉 companion  (double!)
Lv 5  🧭 item        Lv 40 🌈 accessory
Lv 8  🐈 companion   Lv 45 🦄 companion
Lv 10 🗺 item        Lv 50 👑 item + ⭐ accessory  (double!)
Lv 12 🕶 accessory
Lv 15 🦊 companion
Lv 20 🎒 item + 🎀 accessory  (double!)
```

**결과**: Lv 25–45 구간이 `Lv 25 → 30 → 35 → 40 → 45` 매 5레벨마다 해금
이벤트. 중반 슬럼프 해소.

### 마이그레이션 영향
- **신규 아이템 🎒 위치**: Lv 25 → 20 으로 이동. 기존 Lv 25 유저는 이미 해금
  상태라 변화 체감 없음.
- **신규 아이템 ⚡ (Lv 35)**: 기존 Lv 35+ 유저는 즉시 획득 상태 표시.
- **마일스톤 셋 {2,5,10,20,35,50}** — 기존 `_celebratedMilestones` 에
  {25} 가 있던 유저도 그대로 유효 (새 세트에 {25} 없지만 기존값 무시됨).
- **동행 Lv 18 → 15 이동 등**: 기존 Lv 15~17 유저는 🦊 즉시 획득. 기존 Lv 18
  유저는 동일.

## 🎨 Build 125 — 레터 꾸미기: 동행 동물 + 악세사리

유저 정체성이 "레터" 로 확정된 뒤, 레벨업 외에도 **성장을 눈으로 보는**
장치 두 개를 추가. Pokémon GO 의 "버디 포켓몬", Nintendo Animal Crossing
의 의상 커스텀에 해당하는 경량 시스템.

### 1) 동행 동물 (Companion) — 6단계
| 레벨 | 이모지 | 의미 |
|------|--------|------|
| 3 | 🐕 | 강아지 — 첫 동반자 |
| 8 | 🐈 | 고양이 |
| 18 | 🦊 | 여우 |
| 28 | 🦉 | 부엉이 |
| 38 | 🐉 | 드래곤 |
| 48 | 🦄 | 유니콘 — 전설 |

- 14개 언어 모두 "Letter" 그대로 통용 가능
- 앱 정체성 ("편지를 주우러 다니는 사람") 과 직접 정합
- "레터 아이템" = 편지 관련 수집 아이템, 직관적 의미

### 적용 범위 (Build 124)
- l10n key `hunterItemsTitle`:
  - KO: "레터 아이템"
  - EN / FR / DE / ES / PT / IT / TR / AR / HI / TH / RU: "... Letter ..."
  - JA: "レター アイテム"
  - ZH: "Letter 道具"
- 내부 코드 심볼 (`hunterItemEmoji`, `earnedHunterItemLevels`,
  `_buildHunterItems` 등) **변경 없음** — 유저 노출 없는 식별자.

### 이전 후보들 (과거 기록)

- ~~레고 / Lego~~ (Build 123) — The LEGO Group 상표 충돌로 폐기
- 파발꾼 / Herald — 한국 전통 + RPG 감성
- 레터러 / Letterer — 브랜드 확장 신조어

Build 124 의 "Letter" 는 위 후보들을 고려한 후 **가장 단순·안전·브랜드 직결**

Build 123 에서 잠시 "레고 (Lego)" 로 지정했지만 LEGO 상표 리스크로

  - ZH: "Lego 道具"
- 내부 코드 심볼 (`hunterItemEmoji`, `earnedHunterItemLevels`, `_buildHunterItems`
  등) 은 **유지**. 유저 노출 없는 식별자이므로 리팩터 비용을 아낌.

### ⚠️ LEGO 상표 리스크 (중요)

"Lego" 는 **The LEGO Group** (덴마크) 이 전 세계에 걸쳐 등록·행사하는
상표. 특히 아래 시장에서 명칭 충돌 리스크 매우 높음:

- 🇨🇳 중국어 **乐高** = LEGO 공식 중국어 브랜드명 (상표 소유)
- 🇯🇵 일본어 **レゴ** = LEGO 일본 브랜드 (상표 소유)
- 🇺🇸 / 🇪🇺 영문 **LEGO** 모든 대소문자 변형 상표 보호
- 장난감·게임·디지털 앱 카테고리 모두 포괄

### 권장 조치

글로벌 앱스토어 런칭 이전에 **반드시** 상표 변호사 검토 필요. 검토 후
리스크 실체화 시 아래 안전 변형 중 택일:

1. **LeGo** (대소문자 카멜케이스) — 가장 단순
2. **Le:Go** (콜론 삽입) — 타이포그래픽 차별
3. **Le-go** (하이픈) — 문자 분리
4. **LETgo** 또는 **LET-GO** — 대문자 변조
5. **레고어 / Lego-er** — 파생형으로 의미 확장

### 현재 설정은 사용자 결정

본 문서 작성 시점(Build 123) 기준 "Lego" 직접 사용은 사용자가 리스크를
인지하고 결정한 사항. AI 자동화는 법률 자문 대체 아님. 스토어 제출 전
최종 점검 필수.

## 🏷 과거 "헌터" 명칭 대안 5가지 (Build 122 이전)

현재 `hunterItemsTitle` / `towerPulseHint` 등에 "헌터" 단어가 쓰이고 있으나,
다른 이름으로 교체하고 싶다면 아래 중 선택 가능. 선택 후 l10n 키 14개 언어
× ~3–5 문장 재번역 필요.

### 옵션 비교

| # | 이름 (KO / EN) | 느낌 | 장점 | 단점 |
|---|---------------|------|------|------|
| A | **탐험가 / Explorer** | 보편적, 지도 위 이동 | 글로벌 이해 쉬움, 중립 | 쿠폰 느낌 약함 |
| B | **수집가 / Collector** | 주워 모으는 메타포 | 헌트 루프 직결 | 수동적 인상 |
| C | **모험가 / Adventurer** | 젊은 감성, 탐험+발견 | 게임적 톤 | 다소 과장됨 |
| D | **편지꾼 / Letter-person** | 기존 XP 티어와 일관 | 브랜드 정체성 유지 | 서구권에 낯섬 |
| E | **트레저러 / Treasurer** | 보물찾기 직접 | 짧고 영문 통용 | 한국어로 어색 |
| F | **픽커 / Picker** | 활동 직접 번역 | 짧음 | 한국어로 어색 |

### 추천

**1순위: 탐험가 / Explorer**
- 헌트 포지셔닝과 정합 ("지도 위를 탐험")
- 글로벌 7개 대륙 사용자 모두 이해
- "트레저 헌터" 느낌 포함하면서 덜 공격적

**2순위: 수집가 / Collector**
- "주워 모으는" 루프에 가장 직접적
- 바뀐 `huntWalletTitle` "나의 헌트 기록" 과도 자연스럽게 연결

**3순위: 편지꾼 / Letter Collector**
- 기존 XP 티어 라벨과 일관성 ("전설의 편지꾼")
- 브랜드 이름 Thiscount 와 톤 매칭
- 글로벌: "Letter Collector" 로 번역 가능

### 선택에 따른 영향 범위

- `hunterItemsTitle` — "헌터 아이템" → "탐험가 아이템" / "수집가 아이템" 등
- `hunterItemLockedHint` — 변화 없음 (Lv N 해금만)
- `huntWalletTitle` — 변화 없음 (이미 "나의 헌트 기록")
- 문서 내 서술 — 추후 포지셔닝 문서 일괄 갱신
