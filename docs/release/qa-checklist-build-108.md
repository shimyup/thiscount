## 🛡 Firestore 규칙 재점검 — Build 105–108

### 결론: **규칙 변경 불필요** (Build 104 의 결론 그대로)

Build 105–108 에서 추가된 모든 신규 상태는 **로컬 저장소 또는 메모리**
기반이라 Firestore 에 도달하지 않습니다.

| 기능 (빌드) | 저장소 | 규칙 영향 |
|-------------|--------|-----------|
| `_redeemedLetterIds` (108) | SharedPreferences `redeemedLetterIds` | 없음 |
| `_brandExactDropCredits` (106) | SharedPreferences `brandExactDropCredits` | 없음 |
| `_mutedBrandIds` (post-104) | SharedPreferences `mutedBrandIds` | 없음 |
| `_promoShownThisSession` (107) | 메모리 bool | 없음 |
| `userPoints` (106) | XP 로부터 순수 파생 | 없음 |
| 레벨 반경 보너스 (106) | XP 로부터 순수 파생 | 없음 |
| `Letter.redemptionInfo` (post-104) | 기존 `letters/{id}` 문서 필드 | 기존 `letters` 규칙 커버 |
| `Letter.acceptsReplies` (post-104) | 기존 `letters/{id}` 문서 필드 | 기존 `letters` 규칙 커버 |

Build 99 에서 추가된 `lastSeenAt` · `loggedOutAt` 도 기존
`allow write: if isSignedIn()` 하에서 이미 동작 중이며 필드 레벨 규칙
추가는 필요 없습니다.

### 후속 권장 (Build 104 QA 문서 기존 논의 그대로)

- Brand 계정에 한해 **App Check** (reCAPTCHA · DeviceCheck) 도입 검토 —
  쿠폰/교환권 위조 리스크를 한 번 더 줄이는 독립 작업으로 진행
- 서버 측 로그 기반의 ExactDrop 크레딧 소비 이벤트 감사 — 현재는 로컬

  레저가 필요
