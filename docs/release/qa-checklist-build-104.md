## 🤳 실기기 수동 QA (Build 102–104)

### 1. 햅틱 · FeedbackService (Build 102·103)

기기를 "집중·진동 켜짐" 상태로 두고 무음이 아닌 상태로 테스트.

- [ ] **줍기 햅틱** — 근처 편지를 주울 때 가벼운 탭 1회 느껴짐 (Free/Premium)
- [ ] **Brand 줍기 차단** — 브랜드 계정에서 줍기 시도 시 에러 메시지 + 햅틱 없음
- [ ] **3단 개봉 햅틱** — 편지 열 때 3회 다른 세기로 울림
  - Phase 1 (0–400ms): light tap — 봉투 상승 시
  - Phase 2 (400–750ms): medium tap + system click — 실링 버스트
  - Phase 3 (750–1500ms): heavy tap — 편지지 펼침
- [ ] **송신 햅틱** — 편지 보내기 완료 시 성공 햅틱

## 🛡 Firestore 규칙 재점검 — `lastSeenAt` / `loggedOutAt`

### 결론: **규칙 변경 불필요**

현재 `firestore.rules` 의 `/users/{userId}` 쓰기 규칙:

```js
allow write: if isSignedIn();
```

Build 99 에서 추가된 `lastSeenAt` · `loggedOutAt` 필드는 `_saveUserToFirestore()`
→ `FirestoreService.setDocument()` 경유로 PATCH 에 updateMask 와 함께
전송됩니다 (Build 100 수정). 기존 write 규칙이 "서명된 사용자면 모든 필드
쓰기 허용"이므로 두 타임스탬프도 **자동으로 커버**되며 별도 필드 레벨 규칙은
추가할 필요 없습니다.

### 현재 규칙의 알려진 한계 (Build 99 와 무관한 기존 이슈)

`write: if isSignedIn()` 은 **어떤 서명된 사용자든 다른 사용자의 문서를
덮어쓸 수 있음**을 허용합니다. `firestore.rules` 파일 상단 주석이 이미
설명하듯이, 이 앱은 **anonymous Firebase Auth UID 가 세션마다 회전**하기
때문에 표준 `request.auth.uid == userId` 제약을 쓸 수 없습니다.

| 대안 | 장점 | 리스크 |
|------|------|--------|
| 현상 유지 (`isSignedIn()`) | OTP 인증 이후 UID 회전 문제 없음 | 악의적 클라이언트가 다른 사용자 문서 스푸핑 가능 |
| `request.auth.uid == resource.data.authUid` | 소유자만 쓰기 | UID 회전 시 기존 문서 쓰기 불가 → 로그인마다 새 문서 생성 되어 데이터 파편화 |
| Firestore Admin SDK 전용 쓰기 (Cloud Functions 경유) | 서버 측 완전 제어 | 배포 복잡도 증가, 레이턴시 증가 |
| App Check 추가 (reCAPTCHA · DeviceCheck) | 봇 스푸핑 차단 | UID 회전 문제는 그대로 |

**권장**: 지금은 현상 유지. Build 104 의 포지셔닝 전환이 안정화된 이후,
브랜드 계정에 한정해 **App Check** 를 도입해 쿠폰·교환권 위조 리스크를
한 번 더 줄이는 방향을 별도 작업으로 검토.

### 동시에 점검한 내용

- ✅ `letters` 컬렉션에는 새 `category` 필드가 추가되었으나 기존
  `allow update: if isSignedIn()` 로 커버됨
- ✅ 서버 측 카테고리 오버라이드 (Free/Premium → general) 는 **클라이언트
  코드**에서 처리 — Firestore 규칙이 아닌 앱 레이어 가드이므로 이 규칙
  파일과 무관
- ⚠️ 서버 측 가드는 "정상 빌드에서 실행되는 앱"만 보호. 제3자 REST 클라이언트
  를 막는 것은 위 App Check 논의와 같은 맥락

## 📸 스토어 스크린샷 리프레시 계획

Build 104 이후 **스토어에 올라간 기존 스크린샷에서 달라진 영역**:

| 요소 | 이전 스크린샷 | 신규 반영 사항 |
|------|----------------|-----------------|
| 하단 네비 | 중앙 금빛 펄스 FAB + 4개 주변 | **5탭 평면 네비** (탐험/수집첩/보내기/타워/프로필) |
| 편지 개봉 | 2단, 1200ms | **3단, 1500ms** — 중간 실링 버스트 프레임이 새로 찍힘 |
| 지도 배너 | 없음 | **오렌지 "발송 전용" 배너** (Brand 시연용 스크린샷에만) |
| 인박스 힌트 | 없음 | **티일 톤 "🎟 할인권·이벤트 편지를 주워 활용해보세요"** |
| 인박스 필터 | 기본·읽음 등 | **할인권 · 교환권 칩 추가** |
| 브랜드 뱃지 | 일괄 오렌지 🏢 | **카테고리에 따라 색 변경** (할인권/교환권 = 티일) |
| 프로필 | 활동 카드만 | **XP 진행바 + Level N/50 배지** (Free/Premium) |

### 필수 교체 스크린샷 (최우선)

1. **홈·지도 스크린샷** — 5탭 네비가 분명히 보이는 구도로 재촬영
2. **편지 개봉 "한 장"** — 3단 중 Phase 2(실링 버스트 + 흔들림)를 캡처하면 가장 임팩트 있음
3. **인박스 목록** — 새 필터 칩 + 힌트 + 카테고리 뱃지가 한 프레임에 들어가도록 구성
4. **Brand 시연 컷** (있다면) — 오렌지 "발송 전용" 배너 + 정확 좌표 드롭 모드

### 플랫폼별 자산 요구

**Play Console (Android)**
- 필수 2장 / 최대 8장, 16:9 또는 9:16 (1080×1920, 1440×2560 권장)
- 최소 1장에 "Letters travel slowly" + 5탭 네비 동시 노출 권장

**App Store Connect (iOS)**
- iPhone 6.7" (1290×2796) · 6.5" (1284×2778) · 5.5" (1242×2208) 세트
- 최소 3장 / 최대 10장, 현재 업로드된 세트 전부 Build 104 화면으로 재촬영

### 촬영 환경 권장

- **언어 3종** 이상으로 촬영 권장 — 한국어 · 영어 · 일본어 (지도에 다양한 문자 노출)
- **다크 모드 / 라이트 모드 페어링** — 현재 앱이 day-theme 배너를 쓰므로 라이트 모드 권장
- **샘플 데이터** — 테스트 계정에 **편지 5–10통 수신 + 1–2 카테고리 뱃지 포함 + 프로필 Lv 3+** 상태 미리 준비
- **기기 모델**: iPhone 15 Pro (6.7"), iPhone SE (5.5" 대체), Pixel 8 Pro, Galaxy S24

### 교체 **불필요**한 스크린샷 (있다면)

- Premium (Thiscount Premium) 결제 화면 — Build 104 변경 없음

- 설정 · 프로필 상세 (XP 섹션 외)
